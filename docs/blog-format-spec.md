# 블로그 포맷 스펙 (맛집 후기)

이 문서는 `src/lib/export.ts`와 `src/components/TemplatePreview.tsx`의 로직을 그대로 옮긴
것이다. 자유롭게 쓴 네이버 블로그 초안을 이 포맷으로 재작성할 때 기준으로 쓴다.

## 왜 이렇게 정해졌는가

네이버 에디터에 HTML을 붙여넣으면 스타일 대부분이 사라진다. 실제로 살아남는 건 이것뿐이다:

- 굵게 (bold)
- 글자 크기 (font-size)
- 글자 색 (font color)
- 정렬 (text-align)
- 인용구 (`blockquote`)
- 구분선 (`hr`)

배경색·테두리·둥근 모서리 같은 레이아웃 CSS는 붙여넣는 순간 사라진다. 그래서 정보 박스나
총평처럼 시각적으로 구분하고 싶은 덩어리는 전부 인용구로, 표 같은 레이아웃은 전부 텍스트
줄바꿈으로 표현한다. **재작성할 때도 이 6가지 스타일만 쓴다.**

## 자유 초안을 읽을 때: 사진↔텍스트 짝짓기가 핵심

자유롭게 쓴 초안에는 "이건 대표사진이야" 같은 라벨이 없다. 재작성의 기본 동작은:

1. 초안을 위에서부터 순서대로 훑으면서, **사진 하나마다 그 사진을 설명하는 근처 텍스트**
   (보통 사진 바로 앞이나 뒤 문단/문장)를 하나로 묶는다.
2. 그 묶음(사진 + 관련 텍스트)을 원래 등장한 순서 그대로 이어 붙인다: 사진1 + 관련
   텍스트 + 사진2 + 관련 텍스트 + … 반복.
3. 사진이 뭘 찍은 건지(매장/메뉴판/음식/기타) 문맥상 명확하면, 같은 종류가 연달아
   나올 때만 아래 "전체 구조"의 섹션 소제목(`[매장]`/`[메뉴]`/`[음식 & 후기]`)을 붙인다.
   애매하면 소제목 없이 그냥 사진+텍스트 묶음만 순서대로 둔다 — 무리해서
   분류하지 않는다.
4. 제목·정보 인용구·인트로·총평은 초안에 해당 내용이 있을 때만 그 자리(맨 위/사진들
   앞뒤)에 넣는다.
5. **해시태그는 항상 맨 마지막.**

즉 뼈대는 `제목 → (정보/인트로, 있으면) → 사진1+텍스트 → 사진2+텍스트 → … → (총평,
있으면) → 해시태그`다. 아래 "전체 구조"는 이 뼈대에 매장/메뉴/음식 같은 카테고리가
뚜렷할 때 적용하는 좀 더 자세한 버전이다.

## 전체 구조 (위에서 아래 순서, 카테고리가 뚜렷할 때)

1. **제목** — 가게 이름. 24px, 굵게.
2. **정보 인용구** — 방문일 / 영업시간 / 주차 / 가격대 중 값이 있는 것만. 각 줄은
   `**라벨** · 값` 형식(예: `**방문일** · 2026년 7월`), 전부 하나의 인용구로 묶는다.
   값이 하나도 없으면 이 블록 자체를 생략한다.
3. **인트로** — 방문 계기·첫인상 한두 문단. 평문.
4. **대표 사진** 1장.
5. **[매장]** — 구분선(`hr`) + 19px 굵은 소제목 "매장" + 매장 사진들(각 사진 아래 캡션,
   있으면). 매장 사진이 없으면 섹션째 생략.
6. **[메뉴]** — 구분선 + 소제목 "메뉴" + 메뉴판 사진들 + 메뉴 설명(회색 텍스트,
   `#888888`). 사진도 설명도 없으면 생략.
7. **[음식 & 후기]** — 구분선 + 소제목 "음식 & 후기" + 음식 사진들(캡션 포함).
8. **기타 사진** — 소제목 없이 캡션과 함께 마지막에 붙인다. 없으면 생략.
9. **총평 인용구** — 방문 소감 한 줄~한 문단. 정보 인용구와 같은 스타일(따로 라벨 없이
   문장만).
10. **해시태그** — 파란색(`#3366cc`), 맨 마지막 줄. `,` 또는 공백으로 구분된 단어를
    `#단어` 형태로 바꾸고 공백으로 이어붙인다(이미 `#`가 붙어 있으면 중복으로 더 안 붙임).

사진 순서는 항상 **대표 → 매장 → 메뉴판 → 음식 → 기타**다. 각 사진은 해당 섹션에
등장하는 순서 그대로 번호를 매긴다.

## 서식 상세

| 요소 | 스타일 |
|---|---|
| 제목 | `font-size: 24px` + 굵게 |
| 섹션 소제목 (매장/메뉴/음식 & 후기) | `hr` 다음 줄, `font-size: 19px` + 굵게 |
| 정보/총평 인용구 | `blockquote`. 연속된 줄은 하나의 인용구로 합친다 |
| 메뉴 설명 등 보조 텍스트 | 회색 `#888888` |
| 해시태그 | 파란색 `#3366cc` |
| 그 외 본문 | 평문 `<p>` |

## 사진은 반드시 "진짜 파일"로

사진을 `<img src="외부url">` 형태로 붙여넣으면 네이버가 원본 URL을 계속 참조만 하고
자기 서버(사진 라이브러리)에 등록하지 않을 수 있다 — 나중에 원본이 지워지면 글이
깨진다. 재작성 시 사진 위치·순서만 옮기고, **이미 네이버에 실제 업로드된 이미지 요소는
그대로 유지**한다 (파일을 다시 올리거나 URL로 대체하지 않는다).

## 필드 매핑 참고 (`src/types.ts`의 `ReviewFormData`)

자유롭게 쓴 초안에서 아래 슬롯에 해당하는 내용을 찾아 위 구조에 맞게 배치한다.

- `restaurantName` — 제목
- `visitYear` / `visitMonth`, `hours`, `parking`, `priceRange` — 정보 인용구 각 줄
- `intro` — 인트로 문단
- `menuNote` — 메뉴 섹션의 회색 보조 텍스트
- `summary` — 총평 인용구
- `hashtags` — 해시태그 줄
- 사진 5종 역할(`PhotoRole`): `hero`(대표) / `interior`(매장) / `menu`(메뉴판) /
  `food`(음식) / `etc`(기타) — 각 사진이 어느 섹션에 들어갈지는 사진 내용이나 초안의
  문맥으로 판단한다.

## 참고: 공모주(IPO) 글은 다른 포맷

공모주 모드(`src/ipo/`)는 사진 기반이 아니라 카드 이미지(1080×1080 PNG) 6장 기반이라
구조가 다르다. 텍스트 서식 규칙(굵게/색/인용구/구분선만 살아남음)은 동일하지만, 사진
슬롯 대신 카드 1~6이 정해진 순서(표지 → 기업&실적 → 핵심정보 → 청약일정 → 포인트&유의
→ 마무리)로 본문에 박힌다. 자세한 순서는 `src/ipo/export.ts`의 `buildIpoBlocks` 참고.
